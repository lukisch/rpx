# Security Policy

## Reporting a Vulnerability

If you find a security vulnerability, please report it responsibly:

1. Do not open a public issue for security-sensitive details.
2. Use GitHub private vulnerability reporting: https://github.com/entertain-and-more/rpx/security/advisories/new
3. Include a description, steps to reproduce, affected versions, and potential impact.

## Scope

- Local file access through RPX Pro project data (`rpx_pro_data/`)
- Import and parsing of user-provided rule set JSON files
- Desktop application behavior around local media files and session data
- Build and packaging scripts in this repository

RPX Pro is designed as an offline desktop application. Reports about unintended network access, leakage of local data, or unsafe handling of imported files are in scope.

## Response

This is a solo-maintained open-source project, so response times may vary. Critical issues are prioritized. Please allow reasonable time before public disclosure.
